## [Noise infusion banned from Census Bureau statistical products](https://desfontain.es/blog/banning-noise.html) ⭐️ 9.0/10

The U.S. Census Bureau has banned noise infusion, a differential privacy technique, from its statistical products. The policy prohibits using randomness for disclosure avoidance, favoring coarsening and suppression instead. This decision alters the balance between privacy protection and data accuracy in federal statistics, potentially reducing privacy safeguards while improving data utility for researchers and policymakers. It may also undermine public trust in census data confidentiality. The ban targets not only differential privacy but any disclosure avoidance technique involving randomness, explicitly stating that coarsening should be preferred and suppression used only as a last resort. This follows a demonstration of a reconstruction attack on 2010 census data without noise infusion.

**Background**: Differential privacy adds carefully calibrated statistical noise to query results to prevent re-identification of individuals while preserving aggregate patterns. The Census Bureau adopted noise infusion for the 2020 census, but critics argued it degraded data quality. The new policy likely responds to concerns from data users about accuracy and usability.

## [Pyodide 314.0 Allows Direct WASM Wheel Publishing to PyPI](https://simonwillison.net/2026/Jun/13/publishing-wasm-wheels/#atom-everything) ⭐️ 8.0/10

Pyodide 314.0 enables publishing Python packages built for Pyodide (or any runtime compatible with PEP 783's PyEmscripten platform) directly to PyPI as WebAssembly wheels, eliminating the need for Pyodide maintainers to host over 300 packages themselves. Package maintainers can now build and upload WASM wheels just like native Linux, macOS, or Windows wheels. This significantly reduces the maintenance burden on the Pyodide project and removes a major bottleneck for the community, as every new package previously required manual review. It opens the door for a wider ecosystem of Python packages to run efficiently in the browser via Pyodide, potentially accelerating web-based Python applications and tools. The feature relies on PEP 783, which defines the PyEmscripten platform tag for binary Python distributions, and tools like cibuildwheel now support building WASM wheels. An example package, luau-wasm (a WebAssembly build of the Luau language), was published to PyPI as a 276KB wheel and can be installed in Pyodide using micropip.

rss · Simon Willison · Jun 13, 23:55

**Background**: Pyodide is a port of CPython to WebAssembly/Emscripten, enabling Python execution in the browser and Node.js. Previously, Pyodide maintainers had to manually build and host over 300 packages with C/C++ extensions because no standard mechanism existed to distribute WASM wheels on PyPI. PEP 783, accepted recently, standardizes the platform tag for Emscripten-based Python builds, making it possible to upload and install such wheels via regular PyPI infrastructure.
