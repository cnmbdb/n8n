## [Apple 为 macOS 推出容器机器](https://github.com/apple/container/blob/main/docs/container-machine.md) ⭐️ 9.0/10

Apple 在其 Container 工具中引入了容器机器功能，允许开发者在 macOS 上运行符合 OCI 标准的 Linux 容器，支持持久化和文件系统挂载，提供了一个轻量级的 Linux 开发环境。 这项创新为 macOS 开发者提供了一个原生的、轻量级的 Linux 开发替代方案，可能减少对 Docker Desktop 或 Colima 等第三方工具的依赖，并简化工作流程。 容器机器利用 macOS Virtualization.framework 生成轻量级虚拟机来承载 OCI 容器，实现主机与容器之间的无缝文件共享，并允许使用 macOS 原生工具处理 Linux 产物。

hackernews · timsneath · 6月10日 00:29 · [社区讨论](https://news.ycombinator.com/item?id=48469658)

**背景**: OCI（开放容器倡议）是一个 Linux 基金会项目，旨在标准化容器格式和运行时，确保不同容器系统之间的互操作性。Apple 的容器机器基于这一标准，在 macOS 上提供原生的 Linux 容器体验，与传统的虚拟化或基于 Docker 的方法不同。

## [一个错误字符导致 Linux 内核严重漏洞](https://arstechnica.com/security/2026/06/a-single-errant-character-in-the-linux-kernel-allows-attacker-to-gain-root/) ⭐️ 9.0/10

Linux 内核 nf_tables 子系统中一个错误的感叹号引入了一个释放后重用漏洞（CVE-2026-23111），攻击者可利用该漏洞将权限提升至 root 并绕过沙盒防御。 该漏洞极其危险，因为它允许无特权用户获得完全 root 权限并逃逸容器或沙箱环境，影响无数 Linux 服务器和设备。 漏洞位于判定映射删除过程中的引用计数逻辑中；错误的比较导致引用计数被任意递减，引发释放后重用。修复补丁已于 2026 年 2 月合并到 Linux 内核主线，多个发行版已完成修补。

rss · Ars Technica · 6月9日 15:12

**背景**: 释放后重用漏洞发生在程序在内存被释放后仍然引用该内存时，攻击者可借此破坏数据或执行任意代码。nftables 是 Linux 内核中用于包过滤和分类的子系统，广泛应用于防火墙和网络工具。
