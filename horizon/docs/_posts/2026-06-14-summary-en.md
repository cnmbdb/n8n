## [Pyodide 314.0 enables WASM wheels on PyPI](https://simonwillison.net/2026/Jun/13/publishing-wasm-wheels/#atom-everything) ⭐️ 6.0/10

Pyodide 314.0 allows package maintainers to publish WebAssembly (WASM) Python wheels directly to PyPI, using the new pyemscripten platform tag defined in PEP 783. Previously, the Pyodide team had to build and host over 300 packages themselves. This change dramatically reduces the maintenance burden on Pyodide maintainers and removes a major bottleneck for the community. It enables any Python package author to distribute WASM-compiled extensions for browser-based Python runtimes, accelerating the growth of the Python-in-the-browser ecosystem. The new platform tag is pyemscripten_2026_0_wasm32 (versioned by year and patch), and wheels use cibuildwheel for building. The first example package, luau-wasm, demonstrates embedding a C++ Luau interpreter in a 276KB wheel installable via micropip in the browser.

rss · Simon Willison · Jun 13, 23:55

**Background**: Pyodide is a Python runtime for the browser and Node.js based on WebAssembly. Traditionally, Pyodide had to maintain its own package CDN because PyPI did not accept WASM wheels. PEP 783 standardized the Emscripten platform tag, and the PyPI infrastructure (Warehouse) added support, enabling direct publishing.
