## [LLM 发现 FFmpeg 21 个零日漏洞](https://depthfirst.com/research/21-zero-days-in-ffmpeg) ⭐️ 8.0/10

研究人员利用大型语言模型（LLM）发现了 FFmpeg 中的 21 个零日漏洞，其中包括一个可通过攻击者控制的 RTSP URL 利用的严重远程代码执行缺陷。 这项研究展示了 LLM 在自动化漏洞发现方面日益增强的能力，并突显了 FFmpeg 中存在的严重安全风险——该库广泛用于媒体管道、监控系统和转码服务。 其中一个零日漏洞允许 FFmpeg 在处理恶意 RTSP 流时执行远程代码，影响任何接受用户提供的流 URL 的部署。这些漏洞是通过 LLM 辅助的模糊测试和代码分析发现的。

hackernews · redbell · 6月12日 22:13 · [社区讨论](https://news.ycombinator.com/item?id=48510046)

**背景**: FFmpeg 是一个广泛使用的开源多媒体框架，负责音频/视频编解码和流媒体处理。其代码库庞大复杂且用 C 语言编写，历史上因此产生了大量内存损坏漏洞。LLM 越来越多地被用于安全研究中，以生成测试输入并分析代码中的潜在缺陷，从而降低了发现可利用漏洞的门槛。

llama.cpp 版本 b9606 引入了对 EAGLE3 推测解码的支持，该技术通过使用草稿模型生成多个令牌，然后由目标模型并行验证，从而加速 LLM 推理。 这一更新显著改善了 llama.cpp 用户的推理延迟和吞吐量，使实时应用中的响应更快，并提高了本地 LLM 部署的效率可行性。 该版本包含层输入提取、Gemma4 模型集成，并修复了多序列和词汇映射问题。部分功能如 macOS 上的 KleidiAI 和 Ubuntu 上的 SYCL FP32 因待定拉取请求而暂时禁用。

**背景**: llama.cpp 是一个流行的开源 C/C++ 项目，用于在 CPU 和 GPU 上以最少配置本地运行大型语言模型。推测解码是一种加速推理的技术：由一个小型草稿模型提出多个候选令牌，然后由完整模型在一次前向传播中验证，从而减少顺序解码步骤的数量。
